# 💰 Предсказание доходов клиентов банка

Проект направлен на создание высокоточной регрессионной модели, которая оценивает ежемесячный доход клиента на основе его транзакционной активности, социально-демографических данных и кредитной истории.

## 🚀 Ключевые улучшения

По сравнению с базовым решением, данная версия включает:

| Аспект | Исходное решение | **Улучшенное решение** |
| --- | --- | --- |
| **Модели** | 1 модель (последний fold) | **Ансамбль 10 моделей** (2 конфигурации × 5 folds) |
| **Feature Engineering** | ~220 признаков | **~400+ признаков** (агрегации, отношения, логи) |
| **Регуляризация** | Нет L1/L2 | **L1=0.1, L2=0.1** (борьба с переобучением) |
| **Параметры обучения** | LR 0.05, leaves 128 | **LR 0.03, leaves 63** (более стабильный градиент) |
| **Blending** | Нет | **Оптимизация весов** предсказаний |

**Ожидаемый прирост качества:** снижение WMAE на **200–500 единиц**.

## 📈 Основные этапы работы

### 1. Advanced Feature Engineering

Мы расширили признаковое пространство более чем в два раза, добавив:

* **Финансовые индикаторы:** Логарифмирование денежных сумм и вычисление коэффициентов (например, отношение кредитового оборота к дебетовому).
* **Категорийные траты:** Агрегация расходов по группам (продукты, транспорт, развлечения, здоровье).
* **BKI & Digital Profile:** Статистические показатели по данным бюро кредитных историй.
* **Взаимодействия:** Перекрестные признаки, такие как `salary_per_age` или `salary_x_turn`.

### 2. Стратегия валидации

Используется **K-Fold кросс-валидация** на 5 фолдах. Это обеспечивает устойчивость модели к выбросам и позволяет использовать Out-of-Fold (OOF) предсказания для блендинга.

### 3. Ансамблирование (Blending)

Вместо одной модели используется комбинация двух разных конфигураций LightGBM:

1. **LGBM1:** Сбалансированная модель с сильной регуляризацией.
2. **LGBM2:** Более глубокая модель с ограничением `max_depth` для повышения разнообразия (diversity) ансамбля.
